A traditional keyword-based database search is usually not enough for searching large legal contracts using natural language questions. Keyword search works by finding exact words from the query in the document. However, legal documents often use different words or complex language to describe the same idea. For example, a lawyer might ask “What are the termination clauses?” but the contract may use phrases like “conditions for ending the agreement” or “grounds for cancellation.” A keyword search may miss these sections because the exact word “termination” is not used.

Another challenge is that contracts can be very long, sometimes hundreds of pages. The same concept may appear in several places with slightly different wording. Keyword search cannot understand the meaning of a question; it only looks for matching words. Because of this, it may return many irrelevant results and still miss important information. Lawyers would then need to spend extra time reading through the results.

A vector database helps solve this problem by enabling semantic search. In this approach, the contract text is divided into smaller sections and converted into numerical representations called embeddings using a machine learning model. These embeddings capture the meaning of the text.

When a lawyer asks a question in plain English, the system converts the question into an embedding and compares it with the stored embeddings in the database. The system then finds the sections that are most similar in meaning, even if different words are used. This helps lawyers quickly find relevant clauses in large contracts.
